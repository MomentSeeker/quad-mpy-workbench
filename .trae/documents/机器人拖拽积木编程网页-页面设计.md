# 机器人拖拽积木编程网页 - 页面设计（桌面优先 / Scratch 卡通风格）

## 全局设计规范

### Layout
- 桌面优先：≥1200px 为主设计宽度；内容最大宽度 1280px，左右留白。
- 主布局：左右分栏（左：积木库；中：工作区；右：状态/运行/日志），使用 CSS Grid（12 列）+ 局部 Flex。
- 响应式：
  - 992–1199px：右侧面板折叠为可切换 Tab（状态/日志）。
  - ≤991px：改为上下堆叠（积木库可抽屉式），但仍以桌面体验为核心。

### Meta Information（建议）
- Title：机器人积木编程工作台
- Description：像 Scratch 一样拖拽积木，快速控制你的机器人。
- Open Graph：
  - og:title / og:description 同上
  - og:type=website

### Global Styles（Scratch 卡通主题 Tokens）
- 背景：#F6F7FB（浅灰蓝）
- 主色：#FFB000（橙） / 辅色：#4C97FF（蓝） / 成功：#2ECC71 / 危险：#FF4D4F
- 字体：系统字体栈；标题 20/18/16，正文 14/13
- 卡片：圆角 14px；外描边 2px（#2B2B2B，30% 透明）；阴影 0 6px 0 rgba(0,0,0,0.12)
- 按钮：大号胶囊形（圆角 999px），默认高 40px；hover 轻微上浮 1px；active 下压并减弱阴影
- 链接：下划线仅 hover 出现
- 动效：仅短动效（120–180ms），强调“弹性/卡通”但不过度（避免眩晕）

## 页面 1：编程工作台（首页）

### Page Structure
- 顶部导航（固定）
- 主内容三栏：
  1) 左侧「积木库」
  2) 中间「工作区」
  3) 右侧「控制与反馈」

### Sections & Components
1. 顶部导航栏
   - 左：Logo（卡通机器人图标）+ 产品名
   - 中：页面导航（工作台/连接与设置/示例与帮助）
   - 右：连接状态徽章（未连接/已连接/请求中）

2. 左侧：积木库（Palette）
   - 分类 Tab：运动/灯光/声音/控制/逻辑/变量（具体以你定义的积木为准）
   - 搜索框：过滤积木名称
   - 积木列表：大色块卡通积木外观（对应类别色），拖拽提示“可放入工作区”

3. 中间：工作区（Workspace）
   - 网格背景（淡点阵）+ 画布缩放控制（+/-/100%）
   - 拖拽拼接：积木吸附、高亮可连接点
   - 画布工具：撤销/重做、居中、清空（清空需二次确认）

4. 右侧：控制与反馈
   - 运行控制：运行、停止、单步（可选）按钮组；运行前校验提示
   - 请求摘要：显示本次下发目标地址、超时/重试配置摘要
   - 日志面板：按时间列出最近请求结果（成功/失败/耗时）；错误给出可行动建议（例如检查地址、网络、跨域）

### Interaction States
- 未连接：运行按钮置灰，提示“先去连接与设置完成连接测试”
- 请求中：按钮进入 loading；状态徽章显示“请求中”；日志实时追加

## 页面 2：连接与设置

### Page Structure
- 2 列表单布局：左侧连接信息，右侧测试结果与说明

### Sections & Components
1. 连接信息卡
   - 输入：机器人地址（IP/域名）、端口、协议（http/https）
   - 通信策略：超时（ms）、失败重试次数
   - 保存按钮：保存到浏览器本地存储

2. 连接测试卡
   - “开始测试”按钮：发起一次 `POST /control`（按既有协议的安全测试方式）
   - 结果区：显示成功/失败、延迟、错误原因摘要

3. 说明与注意事项
   - 提示：浏览器可能存在跨域限制；建议同域部署或使用反向代理透传（不改机器人端接口）

## 页面 3：示例与帮助

### Page Structure
- 左：示例列表
- 右：示例详情/积木说明（Tab）

### Sections & Components
1. 示例列表
   - 示例卡片：名称、难度标签、简短描述
   - 操作：预览/载入到工作区（载入前提示是否覆盖当前工作区）

2. 积木说明
   - 按类别展示：每个积木的用途、参数说明、常见组合方式（以你的积木定义与机器人协议为准）

3. FAQ
   - 无法连接/超时/机器人无响应/跨域/Cookie 或混合内容（https 页面请求 http 设备）等排查清单
